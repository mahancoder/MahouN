# 📋 گزارش کامل Ultra Contract Agent

## 🎯 خلاصه اجرایی

فایل `ultra_contract_agent.py` به عنوان یکی از مهم‌ترین اجزای سیستم MAHOUN، به طور کامل توسعه یافته و شامل قابلیت‌های پیشرفته تحلیل قرارداد، ارزیابی ریسک بند به بند، و تولید توصیه‌های حقوقی می‌باشد.

## 🏗️ معماری سیستم

```
UltraContractAgent
├── Configuration
│   ├── ContractAgentConfig (تنظیمات RAG, Reasoning, Verification)
│   ├── RiskLevel (5 سطح: CRITICAL, HIGH, MEDIUM, LOW, MINIMAL)
│   └── ClauseType (46 نوع بند قراردادی)
│
├── Data Classes
│   ├── ClauseRisk (ارزیابی ریسک هر بند)
│   ├── ContractAnalysis (تحلیل کل قرارداد)
│   ├── ReasoningStep (گام استدلال)
│   └── ReasoningChain (زنجیره استدلال)
│
├── Core Methods
│   ├── analyze_contract() → تحلیل کامل قرارداد
│   ├── analyze_specific_clause() → تحلیل بند خاص
│   ├── _extract_clauses() → استخراج بندها
│   ├── _analyze_clause() → تحلیل هر بند
│   ├── _detect_clause_type() → تشخیص نوع بند
│   ├── _calculate_clause_risk() → محاسبه ریسک
│   └── _generate_recommendations() → تولید توصیه
│
└── Integration Points
    ├── RAG Service → جستجوی اسناد مرتبط
    ├── Reasoning Service → استدلال گام به گام
    └── NLI Verifier → تأیید پاسخ‌ها
```

## 🧮 الگوریتم محاسبه ریسک

```python
def calculate_risk(clause_text, clause_type):
    risk_score = 0.0
    
    # 1. شاخص‌های ریسک بالا (+0.3 هر کدام)
    for indicator in RISK_INDICATORS[clause_type]["high_risk"]:
        if indicator in clause_text:
            risk_score += 0.3
    
    # 2. شاخص‌های ریسک متوسط (+0.15 هر کدام)
    for indicator in RISK_INDICATORS[clause_type]["medium_risk"]:
        if indicator in clause_text:
            risk_score += 0.15
    
    # 3. الگوهای عمومی ریسک بالا (+0.2 هر کدام)
    general_patterns = ["بدون محدودیت", "نامحدود", "غیرقابل فسخ", "یکطرفه", "فوری"]
    for pattern in general_patterns:
        if pattern in clause_text:
            risk_score += 0.2
    
    # 4. ابهام (+0.1 هر کدام)
    ambiguous = ["ممکن است", "احتمالاً", "در صورت لزوم", "به تشخیص"]
    for term in ambiguous:
        if term in clause_text:
            risk_score += 0.1
    
    # 5. حداقل امتیاز برای بندهای حساس
    if clause_type in [LIABILITY, TERMINATION, PENALTY] and risk_score < 0.2:
        risk_score = 0.2
    
    return min(1.0, risk_score)  # سقف 1.0
```

## 📋 نمونه توصیه‌های تخصصی

### بند پرداخت (PAYMENT)
1. تعیین زمان‌بندی دقیق پرداخت‌ها با تاریخ مشخص
2. اضافه کردن شرایط استرداد وجه در صورت عدم اجرا
3. درج ضمانت‌نامه بانکی یا چک تضمین
4. تعیین نرخ تأخیر تأدیه مطابق قانون
5. مشخص کردن ارز پرداخت و نرخ تبدیل
6. پیش‌بینی تعدیل قیمت در قراردادهای بلندمدت

### بند فسخ (TERMINATION)
1. تعیین مهلت اخطار کتبی قبل از فسخ (حداقل ۳۰ روز)
2. مشخص کردن موارد فسخ فوری (تخلفات اساسی)
3. تعیین تکلیف تعهدات پس از فسخ
4. درج حق اصلاح (Cure Period) قبل از فسخ
5. مشخص کردن نحوه تسویه حساب پس از فسخ
6. تعیین سرنوشت اموال و اسناد پس از خاتمه

### بند مسئولیت (LIABILITY)
1. تعیین سقف مسئولیت (Cap on Liability)
2. استثنا کردن خسارات غیرمستقیم و تبعی
3. محدود کردن مسئولیت به میزان قرارداد
4. تفکیک مسئولیت عمدی از غیرعمدی
5. درج بیمه مسئولیت حرفه‌ای
6. تعیین مهلت اعلام خسارت
