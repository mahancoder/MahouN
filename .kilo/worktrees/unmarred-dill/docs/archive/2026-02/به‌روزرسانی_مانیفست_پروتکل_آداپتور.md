## الگوی معماری

**الگو**: الگوی Orchestrator با تزریق وابستگی مبتنی بر پروتکل

### مزایا:
1. ✅ ماژول‌های هسته خالص می‌مانند - بدون وارد کردن از غیر هسته
2. ✅ عملکرد غیر هسته اختیاری و قابل تعویض است
3. ✅ تست ماژول‌های هسته به صورت مجزا آسان است
4. ✅ جداسازی واضح نگرانی‌ها
5. ✅ از اصول SOLID پیروی می‌کند (وارونگی وابستگی)

## مثال استفاده

### بدون تزریق وابستگی (فقط هسته):
```python
from mahoun.reasoning import EvidenceLinkedVerdictEngine
engine = EvidenceLinkedVerdictEngine()
```

### با تزریق وابستگی (هسته + غیر هسته):
```python
from mahoun.reasoning.adapters import (
    create_reasoning_engine,
    create_query_router
)
from mahoun.core.llm.orchestrator import ModelOrchestrator

router = create_query_router()
orchestrator = ModelOrchestrator()
engine = create_reasoning_engine(
    query_router=router,
    model_orchestrator=orchestrator
)
```
