## 1. چیزی که من دیدم (نگاه فنی)

### تفاوت اصلی ماحون با سیستم‌های معمولی

بیشتر سیستم‌های AI که می‌بینم این شکلی هستن:
```
User Question → ChatGPT API → Answer
```

مشکل: نمی‌دونی جواب از کجا اومده، ممکنه اشتباه باشه، قابل audit نیست.

ماحون اینطوری کار می‌کنه:
```
User Question 
  ↓
Knowledge Graph (اسناد واقعی شما)
  ↓
Evidence Extraction (استخراج شواهد)
  ↓
Reasoning Engine (استدلال مبتنی بر شواهد)
  ↓
Evidence Ledger (ثبت غیرقابل تغییر)
  ↓
Answer + Proof (جواب + مدرک)
```

این یعنی: هر جواب یک "زنجیره شواهد" داره که قابل بررسی است.

### نقاط قوت معماری (چیزهایی که تحسین‌برانگیزه)

#### ✅ معماری Zero-Hallucination واقعی
```
سیستم از Knowledge Graph استفاده می‌کنه که:
- هر جواب به یک سند واقعی لینک می‌شه
- نمی‌تونه چیزی از خودش درآورد
- همه چیز قابل ردیابی و audit هست
```

این یعنی چی؟ یعنی وقتی سیستم می‌گه "طبق بند 3.2 قرارداد..."، واقعاً اون بند رو داره و می‌تونی چک کنی.

**مثال واقعی**:
```python
# سیستم معمولی (ChatGPT)
Q: "آیا می‌توانم قرارداد را فسخ کنم؟"
A: "بله، طبق شرایط عمومی می‌توانید..."
❌ نمی‌دونی این جواب از کجا اومده

# ماحون
Q: "آیا می‌توانم قرارداد را فسخ کنم؟"
A: "بله، طبق بند 12.3 قرارداد شماره 1234..."
   Evidence Chain:
   ├─ Document: contract_1234.pdf
   ├─ Section: 12.3
   ├─ Text: "کارفرما می‌تواند در صورت تأخیر بیش از 30 روز..."
   ├─ Confidence: 95%
   └─ Ledger ID: 0x7f3a... (غیرقابل تغییر)
✅ می‌تونی هر مرحله رو چک کنی
```

این برای دادگاه، ممیزی، یا تصمیمات حیاتی طلاست.

#### ✅ Evidence Ledger (دفتر ثبت شواهد)
```python
# هر تصمیم ثبت می‌شه و غیرقابل تغییره
mahoun/ledger/
├── models.py          # مدل‌های داده
├── writer.py          # ثبت شواهد
├── guards.py          # محافظت از یکپارچگی
└── storage.py         # ذخیره‌سازی امن
```

این برای صنایع regulated (بانک، بیمه، دارو) طلاست. می‌تونی 5 سال بعد بگی "چرا این تصمیم گرفته شد؟"

**مثال کاربردی**:
```
سناریو: یک بانک وام 500 میلیون تومانی رد کرده

5 سال بعد، مشتری شکایت می‌کنه:
"چرا وام من رد شد؟ تبعیض بوده!"

بدون Ledger:
❌ نمی‌دونیم چرا رد شد
❌ مدرکی نداریم
❌ ممکنه دعوا ببازیم

با Ledger ماحون:
✅ Ledger Entry #12345 نشون می‌ده:
   - تاریخ: 1399/05/15
   - دلیل: نسبت بدهی به درآمد 65% (حد مجاز 50%)
   - سند: فیش حقوقی مشتری
   - قانون: بخشنامه بانک مرکزی شماره 234
   - تصمیم‌گیرنده: سیستم خودکار (بدون دخالت انسان)
✅ دعوا رو می‌بریم
```

#### ✅ Multi-Model Orchestration هوشمند
```python
# سیستم خودش تشخیص می‌ده کدوم مدل رو استفاده کنه
ModelCapability.CODING → Qwen-Coder (سریع)
ModelCapability.REASONING → Granite-Legal (دقیق)
ModelCapability.GENERAL → Llama (همه‌کاره)
```

این یعنی هزینه بهینه: مدل گرون رو فقط وقتی لازمه استفاده می‌کنه.

**مثال واقعی**:
```python
# سوال ساده: "قرارداد شماره 1234 چه تاریخی امضا شده؟"
→ Qwen-Coder (سریع، ارزان)
→ پاسخ در 0.5 ثانیه
→ هزینه: ~100 تومان

# سوال پیچیده: "آیا طبق قوانین آیین دادرسی، این قرارداد قابل فسخ است؟"
→ Granite-Legal (دقیق، گران‌تر)
→ پاسخ در 3 ثانیه
→ هزینه: ~500 تومان

# سوال خیلی پیچیده: "تحلیل کامل ریسک‌های حقوقی این پروژه"
→ Llama-70B (خیلی دقیق، خیلی گران)
→ پاسخ در 15 ثانیه
→ هزینه: ~2000 تومان
```

این routing خودکار در ماه می‌تونه 40-50% هزینه رو کاهش بده.

#### ✅ Enterprise-Grade Infrastructure
```
✓ Distributed Locking (برای همزمانی)
✓ Circuit Breakers (برای مقاومت در برابر خرابی)
✓ Comprehensive Monitoring (Prometheus + Grafana)
✓ Security Layers (Encryption, Signing, Auth)

```

این چیزهایی نیست که تو یک MVP ببینی. این سطح enterprise است.

```python
# مثال از mahoun/reasoning/evidence_linked_verdict.py
class EvidenceLinkedVerdict:
    """
    Core reasoning engine with 100% groundedness guarantee.
    
    Invariant I1: Every reasoning step MUST link to graph evidence.
    """
    
    def reason(self, query: str) -> Verdict:
        # 1. Extract evidence from knowledge graph
        evidence = self.graph.find_evidence(query)
        
        # 2. Verify evidence exists (I1 enforcement)
        if not evidence:
            raise NoEvidenceError("Cannot reason without evidence")
        
        # 3. Build reasoning chain
        chain = self.build_chain(evidence)
        
        # 4. Record in immutable ledger
        ledger_id = self.ledger.record(chain)
        
        # 5. Return verdict with proof
        return Verdict(
            answer=chain.conclusion,
            evidence=evidence,
            ledger_id=ledger_id,
            confidence=chain.confidence
        )
```

این کد نشون می‌ده:
- ✅ تفکر واضح و منطقی
